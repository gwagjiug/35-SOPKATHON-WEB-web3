### ✅ 커밋 컨벤션과 브랜치 전략

| 유형       | 설명                                   | 예시                          |
|------------|----------------------------------------|-------------------------------|
| `feat`     | 새로운 기능 추가                      |                               |
| `fix`      | 버그 수정                              |                               |
| `chore`    | 빌드 업무, 패키지 매니저, 라이브러리, dependencies 설정 |                               |
| `docs`     | 문서 수정                              | ex) README.md                 |
| `design`   | 사용자 UI 디자인 변경                 | ex) CSS                       |
| `style`    | 기능 수정 없는 코드 스타일 변경       |                               |
| `rename`   | 이름 변경                              |                               |
